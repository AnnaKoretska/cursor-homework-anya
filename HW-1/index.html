Команди які найчастіше використовуються в GIT

  • git config --global user.name blabla - налаштування імені користувача
  • git config --global user.email blabal@bla - налаштування електронної адреси
  • git init-створення нового репозиторію
  • git status-перегляд статусу
  • git add-додавання файлів в репозиторій.Зазвичай використовується «.» або «*» для додавання усіх файлів.
  • git commit -збереження змін. Можна вказати без аргументу -m, в такому випадку відкриється текстовий редактор по замовчуванню, в якому потрібно буде описати коміт. Якщо це сталось випадково, вводите :q і натискаєте Enter
  • git log-перегляд історії змін файлів проекту
  • git checkout-дозволяє переміщатися між гілками, створеними командою git branch.
  • git branch-це команда для керування гілками в репозиторії Git.
  • git push-внесення змін з віддаленого репозиторію в локальний.
  • git pull -отримання змін з репозиторію.
  • git merge-завантаження історії з віддаленого репозиторію і об'єднання з локальною
  • git clone-створення нового локального репозиторію із заданим ім'ям

.gitignore файл

.gitignore файл - потрібен для приховування файлів та папок від системи контролю версій Git.
